# 📱 StoreOps — 교보문고 매장 운영 모바일 앱
### 중간 결과 보고 | 잠실점 | 2026년 6월

---

## 1. 한 줄 요약

> **PDA 기기 없이 직원 스마트폰으로 도서 바코드를 스캔해 재고·서가 위치·발주 현황을 즉시 확인할 수 있는 매장 운영 앱입니다.**

---

## 2. 왜 만들었나요?

| 현재 문제 | 해결 방향 |
|-----------|-----------|
| 재고 확인·서가 위치 조회 시 PDA 기기가 필요함 | 직원 스마트폰으로 즉시 조회 가능 |
| PDA 기기 수량이 한정적이라 대기가 발생함 | 개인 폰 설치로 인원 제한 없음 |
| PDA가 느리고 분실·파손 위험이 있음 | 앱 형태(PWA)로 항상 휴대 가능 |
| 인수인계 내용이 구두·메모 수준에 머뭄 | 앱 내 인수인계 기능으로 체계화 |

> **현재 교보 내부 재고 API 연동 전 단계** — 실제 재고 데이터 대신 샘플(Mock) 데이터로 프로토타입 구현 중

---

## 3. 무엇을 만들었나요?

### 3-1. 직원 화면

#### 🔷 도서 조회 (핵심 기능)
- 스마트폰 카메라로 책 바코드(EAN-13) 스캔 → 도서 정보 즉시 표시
- **제목·ISBN 검색** 지원 (네이버 도서 API 실시간 연동)
- 도서 상세: 재고 수량(정상/예약/불량), 서가 위치(층·서가번호), 발주 현황

#### 🔶 발주 관리
- 발주 목록 조회 및 상태별 필터링

| 상태 | 설명 |
|------|------|
| 대기 | 발주 접수, 처리 전 |
| 승인 | 본사 승인 완료 |
| 완료 | 입고 처리 완료 |

- 담당자별·도서별 발주 현황 확인

#### 🔷 인수인계
- 근무 메모 작성 및 공유
- **공개 범위 설정:** 전체 / 같은 층 직원만 / 관리자만

---

### 3-2. 관리자 화면 (admin 전용)

| 기능 | 설명 |
|------|------|
| 전 층 조회 | 층 제한 없이 전 매장 재고·서가 확인 |
| 직원 관리 | 권한(staff/admin) 및 담당 층 설정 |
| 발주 승인 | 직원 발주 요청 검토 및 승인 처리 |
| 인수인계 전체 열람 | 공개 범위 무관하게 전체 메모 열람 |

## 4. 어떻게 만들었나요?

### 기술 스택

| 항목 | 선택 | 이유 |
|------|------|------|
| 프런트엔드 | HTML / CSS / Vanilla JS (단일 파일) | 설치 없이 브라우저에서 바로 실행 |
| 도서 API | 네이버 도서 검색 API | 제목·ISBN 실시간 검색 |
| CORS 우회 | Python 프록시 서버 (port 8080) | 브라우저 보안 정책 우회 |
| HTTPS 터널 | ngrok | iOS Safari 카메라 허용 필수 조건 |
| 바코드 인식 | ZXing 라이브러리 (EAN-13) | 노트북·스마트폰 모두 동작 확인 |
| 앱 설치 | PWA (manifest + Service Worker) | 홈 화면 아이콘으로 앱처럼 설치 |

### 어댑터 패턴 — API 교체 유연성

```
BookService
  ├── NaverAdapter   ← 현재 사용 (네이버 도서 API)
  ├── KyoboAdapter   ← 교보 내부 API 연동 시 교체 예정
  └── MockAdapter    ← 오프라인 테스트용 샘플 데이터
```

> 교보 자체 API가 생기면 `BookService.use(KyoboAdapter)` 한 줄로 전환 가능

### 개발 중 해결한 주요 문제

| 문제 | 원인 | 해결 |
|------|------|------|
| iOS Safari 카메라 안 켜짐 | HTTPS 필수 환경인데 HTTP로 실행 | ngrok으로 HTTPS 터널 구성 |
| 바코드 인식 안 됨 | JS 문법 오류로 스캔 루프 미동작 | ZXing 동기 API로 교체 |
| 모바일 검색 결과 없음 | 서버 주소가 `localhost`로 하드코딩됨 | 접속 URL 기반 동적 처리 |
| ngrok 경고 페이지 반환 | 무료 플랜 브라우저 경고 HTML | 전용 헤더(`ngrok-skip-browser-warning`) 추가 |
| 앱 업데이트 미반영 | Service Worker가 앱 파일 캐시 | 앱 파일 캐시 제외 처리 |

| PWA 홈 화면 설치 | ✅ 완료 | iOS Safari / Android Chrome |

## 6. 접속 방법

| 구분 | 내용 |
|------|------|
| **접속 방식** | PC와 같은 와이파이에서 ngrok HTTPS URL 접속 |
| **ngrok 실행 중 URL 예시** | `https://xxxx-xxxx.ngrok-free.app/app.html` |
| **관리자 계정** | 사번: `13165` / 비밀번호: `13165` |
| **직원 계정 예시** | 사번: `EMP-10234` / 비밀번호: `admin1234` |

> 📱 PC·스마트폰 모두 접속 가능 / 스마트폰은 홈 화면에 앱으로 설치 가능

## 7. 지금 당장 해볼 수 있는 테스트

1. ngrok URL로 스마트폰 접속 후 사번 `13165` / 비밀번호 `13165` 로 로그인
2. 하단 **스캔** 버튼 탭 → 카메라 켜지면 책 바코드 가져다 대기
3. 도서 정보·재고·서가 위치 확인
4. **검색** 탭에서 책 제목 또는 ISBN 직접 입력해 검색
5. **발주** 탭에서 상태별 필터 탭 전환 (대기 / 승인 / 완료)
6. **인수인계** 탭에서 메모 작성 후 공개 범위 설정
